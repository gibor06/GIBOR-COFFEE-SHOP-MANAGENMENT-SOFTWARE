# Output Template

Use this structure as the default final response.

## A. Executive summary
- current objective
- inferred stack
- overall status

## B. Assumptions
List each technical or project assumption explicitly.

## C. Integrated backlog from project commander
For each major task include:
- task id
- title
- primary agent
- supporting agent
- output
- priority
- milestone
- dependency
- completion criteria

## D. UI/auth plan from agent 1
Include:
- screen inventory
- navigation flow
- login flow
- role matrix
- view/viewmodel/service map
- integration order
- ui test checklist

## E. Data/core plan from agent 2
Include:
- database overview
- table and key design
- CRUD scope
- search rules
- purchase and stock rules
- validation rules
- data test cases

## F. Sales/report plan from agent 3
Include:
- sales flow
- stock deduction logic
- analytics scope
- simple report scope
- advanced report scope
- documentation outline
- demo and screenshot checklist

## G. Commander integration summary
Include:
- build order
- integration order
- system test order
- demo order
- top dependencies

## H. Final delivery package
Include:
- source structure
- database structure
- report and documentation bundle
- demo dataset
- submission checklist

## I. Risks and lock plan
List the top blockers and how to close each one.
